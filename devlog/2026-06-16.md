## GitHub Pages 배포 환경 구축

### 1. Vite base 경로 설정
GitHub Pages는 서브 경로(`/Tomorrower/`)로 서빙되므로 Vite에 base 지정.

```js
// vite.config.js
export default defineConfig({
  plugins: [react(), tailwindcss()],
  base: '/Tomorrower/',
})
```

### 2. React Router basename 추가
`BrowserRouter`가 서브 경로를 인식하도록 `basename` 설정.

```jsx
// src/App.jsx
<BrowserRouter basename="/Tomorrower">
```

### 3. SPA 라우팅 처리 (404.html 트릭)
GitHub Pages는 정적 파일 서버라 `/schedule`, `/info` 등 직접 URL 접근 시 404 반환.  
이를 우회하기 위한 표준 기법 적용.

**`public/404.html`** — 404 시 경로를 sessionStorage에 저장 후 루트로 리다이렉트

```html
<script>
  sessionStorage.setItem('spa-redirect', window.location.pathname + ...);
  window.location.replace('/Tomorrower/');
</script>
```

**`index.html`** — 루트 진입 시 sessionStorage에서 경로를 복원해 history에 적용

```html
<script>
  (function () {
    var redirect = sessionStorage.getItem('spa-redirect');
    if (redirect) {
      sessionStorage.removeItem('spa-redirect');
      window.history.replaceState(null, null, redirect);
    }
  })();
</script>
```

### 4. package.json — homepage 및 배포 스크립트 추가

```json
{
  "homepage": "https://alicelimti.github.io/Tomorrower",
  "scripts": {
    "predeploy": "npm run build",
    "deploy": "gh-pages -d dist"
  }
}
```

- `gh-pages` 패키지 설치 (`devDependencies`)
- `npm run deploy`로 수동 배포도 가능

### 5. OG 메타 URL 변경
`index.html` OG 태그의 URL·이미지 경로를 GitHub Pages 주소로 일괄 수정.

```
https://tomorrower.vercel.app → https://alicelimti.github.io/Tomorrower
```

## GitHub Actions 워크플로우

`.github/workflows/deploy.yml` 생성.  
`main` 브랜치 push 시 자동으로 빌드 → `gh-pages` 브랜치 배포.

```yaml
name: Deploy to GitHub Pages

on:
  push:
    branches: [main]
  workflow_dispatch:

permissions:
  contents: write

jobs:
  deploy:
    runs-on: ubuntu-latest
    steps:
      - uses: actions/checkout@v4
      - uses: actions/setup-node@v4
        with:
          node-version: 20
          cache: npm
      - run: npm ci
      - run: npm run build
      - uses: peaceiris/actions-gh-pages@v4
        with:
          github_token: ${{ secrets.GITHUB_TOKEN }}
          publish_dir: ./dist
```

### 트러블슈팅
- 첫 실행 시 `Deploy to gh-pages branch` 스텝 실패
- 원인: 워크플로우에 `permissions: contents: write` 누락
- 수정 후 재실행 → 성공, `gh-pages` 브랜치 자동 생성 확인

## 내 시험 D-Day 및 시험 관리 기능 추가

### 배경
홈 화면 퀴즈 배너가 실제 사용 맥락과 맞지 않음.  
사용자가 신청한 시험의 D-Day를 첫 화면에서 바로 확인할 수 있도록 개편.

### 구현 내용

#### `src/hooks/useMyExams.js` 신규 생성
- `localStorage` 기반으로 선택한 시험 목록을 영구 저장
- `myExams` (배열) + `toggle(examId)` 반환
- 앱 재시작 후에도 선택 상태 유지

#### 홈 화면 — 퀴즈 배너 → 내 시험 D-Day 카드
- 등록된 시험이 있으면 각 시험의 D-Day 카드 표시
  - 시험명 · 카테고리 배지 · 시험일 · D-Day 뱃지
  - D-7 이내 → 분홍색 강조
  - 상시시험 → 초록 "상시" 뱃지
- 등록된 시험이 없으면 안내 카드 표시 (시험 일정 탭 링크)
- 우측 상단 "시험 추가 →" 링크로 일정 탭으로 바로 이동 가능

#### 시험 일정 탭 — 내 시험 관리 섹션 추가
- 상단에 전체 시험 버튼 목록 표시
- 탭하면 추가(✓ 표시 + 색상), 다시 탭하면 제거
- 추가된 시험은 홈 D-Day 카드에 즉시 반영

### 변경 파일
- `src/hooks/useMyExams.js` — 신규
- `src/pages/Home/index.jsx` — 퀴즈 배너 제거, D-Day 섹션 추가
- `src/pages/Schedule/index.jsx` — 내 시험 관리 섹션 추가
