## GitHub Pages 배포 환경 구축

### 1. Vite base 경로 설정
GitHub Pages는 서브 경로(`/Tomorrower/`)로 서빙되므로 Vite에 base 지정.

```js
// vite.config.js
export default defineConfig({
  plugins: [react(), tailwindcss()],
  base: '/Tomorrower/',
})
```

### 2. React Router basename 추가
`BrowserRouter`가 서브 경로를 인식하도록 `basename` 설정.

```jsx
// src/App.jsx
<BrowserRouter basename="/Tomorrower">
```

### 3. SPA 라우팅 처리 (404.html 트릭)
GitHub Pages는 정적 파일 서버라 `/schedule`, `/info` 등 직접 URL 접근 시 404 반환.  
이를 우회하기 위한 표준 기법 적용.

**`public/404.html`** — 404 시 경로를 sessionStorage에 저장 후 루트로 리다이렉트

```html
<script>
  sessionStorage.setItem('spa-redirect', window.location.pathname + ...);
  window.location.replace('/Tomorrower/');
</script>
```

**`index.html`** — 루트 진입 시 sessionStorage에서 경로를 복원해 history에 적용

```html
<script>
  (function () {
    var redirect = sessionStorage.getItem('spa-redirect');
    if (redirect) {
      sessionStorage.removeItem('spa-redirect');
      window.history.replaceState(null, null, redirect);
    }
  })();
</script>
```

### 4. package.json — homepage 및 배포 스크립트 추가

```json
{
  "homepage": "https://alicelimti.github.io/Tomorrower",
  "scripts": {
    "predeploy": "npm run build",
    "deploy": "gh-pages -d dist"
  }
}
```

- `gh-pages` 패키지 설치 (`devDependencies`)
- `npm run deploy`로 수동 배포도 가능

### 5. OG 메타 URL 변경
`index.html` OG 태그의 URL·이미지 경로를 GitHub Pages 주소로 일괄 수정.

```
https://tomorrower.vercel.app → https://alicelimti.github.io/Tomorrower
```

## GitHub Actions 워크플로우

`.github/workflows/deploy.yml` 생성.  
`main` 브랜치 push 시 자동으로 빌드 → `gh-pages` 브랜치 배포.

```yaml
name: Deploy to GitHub Pages

on:
  push:
    branches: [main]
  workflow_dispatch:

permissions:
  contents: write

jobs:
  deploy:
    runs-on: ubuntu-latest
    steps:
      - uses: actions/checkout@v4
      - uses: actions/setup-node@v4
        with:
          node-version: 20
          cache: npm
      - run: npm ci
      - run: npm run build
      - uses: peaceiris/actions-gh-pages@v4
        with:
          github_token: ${{ secrets.GITHUB_TOKEN }}
          publish_dir: ./dist
```

### 트러블슈팅
- 첫 실행 시 `Deploy to gh-pages branch` 스텝 실패
- 원인: 워크플로우에 `permissions: contents: write` 누락
- 수정 후 재실행 → 성공, `gh-pages` 브랜치 자동 생성 확인
