# 2026-06-19 — Supabase Auth Google 로그인 구현

## 작업 개요

My 탭 시험 목록을 클라우드에 저장하기 위해  
Supabase Auth Google OAuth를 연동하고, `user_exams` 테이블을  
`device_id` 기반에서 `user_id` 기반으로 마이그레이션.

---

## 아키텍처

```
AuthContext (전역)
  └─ useAuth()  →  My 페이지, useMyExams
       ├─ user: 로그인된 사용자 객체
       ├─ authLoading: 세션 로드 중 여부
       ├─ signInWithGoogle(): Google OAuth 팝업
       └─ signOut(): 로그아웃
```

## Supabase Auth 설정

### Google OAuth 설정 (대시보드)
1. **Authentication → Providers → Google** 활성화
2. **Authorized redirect URIs** 에 `https://brwtobsmjyfsbhowhsmg.supabase.co/auth/v1/callback` 등록
3. **URL Configuration → Redirect URLs** 에 `https://alicelimti.github.io/Tomorrower/` 등록
4. Google Cloud Console에서 OAuth Client ID / Secret 발급 후 입력

## user_exams 테이블 마이그레이션

`device_id` (익명 UUID) → `user_id` (Supabase Auth UUID) 로 변경.  
기존 테이블을 drop & recreate로 재구성.

```sql
drop table if exists user_exams cascade;

create table user_exams (
  id         uuid        default gen_random_uuid() primary key,
  user_id    uuid        not null references auth.users(id) on delete cascade,
  exam_id    text        not null,
  created_at timestamptz default now(),
  unique (user_id, exam_id)
);

create index idx_user_exams_user_id on user_exams (user_id);

alter table user_exams enable row level security;

-- 본인 데이터만 접근 가능
create policy "users_own_data" on user_exams
  for all
  using (auth.uid() = user_id)
  with check (auth.uid() = user_id);
```

마이그레이션 파일: `supabase/migration_001_user_id.sql`

## 구현 내용

### `src/contexts/AuthContext.jsx` (신규)

```jsx
export function AuthProvider({ children }) { ... }
export function useAuth() { return useContext(AuthContext); }
```

- 앱 시작 시 `getSession()`으로 기존 세션 복원
- `onAuthStateChange()`로 로그인/로그아웃 상태 실시간 감지
- `signInWithGoogle()`: `signInWithOAuth({ provider: 'google' })`
- `redirectTo`: `window.location.origin + '/Tomorrower/'` (로컬/배포 환경 자동 대응)

### `src/App.jsx`

`<AuthProvider>`로 전체 앱 감싸기

### `src/hooks/useMyExams.js`

- `useAuth()`에서 `user` 가져와 `user.id` 기반으로 Supabase 조회
- 비로그인 시 localStorage 전용으로 동작
- `useDeviceId.js` 삭제 (user_id로 완전 대체)

### `src/pages/My/index.jsx`

| 상태 | UI |
|---|---|
| 로그아웃 | ☁️ Google 로그인 유도 카드 + "이 기기에만 저장" 안내 |
| 로그인 중 | 헤더에 Google 이름 + ✕ 로그아웃 버튼 |
| 동기화 중 | "클라우드에서 불러오는 중..." |

## 트러블슈팅

### 흰 화면 (배포 환경)

**원인**: `.env.local`은 gitignore 처리되어 GitHub Actions 빌드 시  
`VITE_SUPABASE_URL`, `VITE_SUPABASE_ANON_KEY`가 `undefined` →  
`createClient(undefined, undefined)` 에러 → 앱 전체 렌더링 실패

**수정**: `deploy.yml` Build 스텝에 환경변수 주입 + GitHub Repository Secrets 등록

```yaml
- name: Build
  run: npm run build
  env:
    VITE_SUPABASE_URL: ${{ secrets.VITE_SUPABASE_URL }}
    VITE_SUPABASE_ANON_KEY: ${{ secrets.VITE_SUPABASE_ANON_KEY }}
```

### user_exams 테이블 마이그레이션 오류

**원인**: 기존 `device_id` 컬럼이 있는 테이블에 migration SQL 실행 시  
`ERROR: 42703: column "device_id" does not exist` 발생  
→ `DROP TABLE` 후 `CASCADE` 없이 재생성 시 잔여 객체 충돌

**수정**: `drop table if exists user_exams cascade;` 로 수정하여 해결
