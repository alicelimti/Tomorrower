# 2026-06-18 — Firebase → Supabase 교체 및 PWA 구축

## Firebase → Supabase 교체

### 배경
Firebase는 환경 변수 6개, SDK 85개 패키지가 필요한 반면  
Supabase는 환경 변수 2개, 단일 패키지로 Auth + DB 모두 지원.  
아직 실제 연동 코드가 없는 시점에서 선제적으로 교체.

### 변경 내용

| 항목 | 이전 (Firebase) | 이후 (Supabase) |
|---|---|---|
| 패키지 | `firebase` (85개) | `@supabase/supabase-js` |
| 설정 파일 | `src/firebase/config.js` | `src/supabase/client.js` |
| 환경변수 | `VITE_FIREBASE_*` × 6 | `VITE_SUPABASE_URL`, `VITE_SUPABASE_ANON_KEY` |

### `src/supabase/client.js`

```js
import { createClient } from '@supabase/supabase-js';

export const supabase = createClient(
  import.meta.env.VITE_SUPABASE_URL,
  import.meta.env.VITE_SUPABASE_ANON_KEY
);

export const auth = supabase.auth;
export const db = (table) => supabase.from(table);
```

### `.env.local` 설정 완료
- Supabase 프로젝트 생성 후 URL·Anon Key 입력
- 연결 테스트 통과 (HTTP 200 응답 확인)

### 변경 파일
- `src/firebase/config.js` → 삭제
- `src/supabase/client.js` → 신규
- `.env.local` → `VITE_SUPABASE_*` 로 교체 (로컬 전용, gitignore)
- `package.json` → firebase 제거, @supabase/supabase-js 추가

## PWA 업데이트 알림

### 배경
서비스 워커 특성상 새 버전 배포 후 기존 캐시가 남아  
사용자가 최신 버전을 못 받는 문제를 UX로 해결.

### 구현
- `src/components/UpdatePrompt.jsx` 신규 생성
- `vite-plugin-pwa`의 `useRegisterSW` 훅 사용
- 새 서비스 워커 감지 시 하단 탭 위에 토스트 알림 표시

```
┌─────────────────────────────────────────┐
│ 새 버전이 있습니다          나중에  새로고침 │
│ 새로고침하면 최신 버전으로 업데이트돼요      │
└─────────────────────────────────────────┘
```

- **새로고침**: 서비스 워커 즉시 활성화 + 페이지 리로드
- **나중에**: 알림 닫기 (다음 방문 시 재표시)

### 변경 파일
- `src/components/UpdatePrompt.jsx` — 신규
- `src/App.jsx` — `<UpdatePrompt />` 추가

## PWA 아이콘 텍스트 버전으로 교체

### 변경 내용
- 기존 SVG 로고 기반 아이콘 → 텍스트 기반 아이콘으로 교체
- 배경: 라벤더(`#7B75E8`) → 핑크(`#E8A0D0`) 대각선 그라디언트
- 중앙 흰색 굵은 글씨 **"미루지 / 말자"** 두 줄

| 파일 | 크기 |
|---|---|
| `public/pwa-192x192.png` | 192×192 |
| `public/pwa-512x512.png` | 512×512 (maskable 겸용) |

### 홈 화면 아이콘 갱신 방법
OS가 설치 시점에 아이콘을 고정 저장하므로 **앱 삭제 후 재설치** 필요.  
이후 콘텐츠(JS/CSS) 업데이트는 "새로고침" 알림으로 자동 반영.

## 트러블슈팅 — PWA 업데이트 알림 미작동

### 원인
`vite.config.js`의 `registerType: 'autoUpdate'`와 `UpdatePrompt`(`useRegisterSW`) 충돌.  
`autoUpdate` 모드에서는 `needRefresh` 상태가 트리거되지 않아 알림이 표시되지 않음.

### 수정
`registerType: 'autoUpdate'` → `'prompt'` 로 변경.  
이제 새 서비스 워커 감지 시 `needRefresh = true` → 하단 토스트 정상 표시.

### 추가했던 내용

로그인 기능 구현 전까지 보류하기로 결정.

### 제거된 파일 및 코드

## 내 시험 목록 Supabase DB 연동

### 배경
My 탭의 시험 추가/삭제가 localStorage에만 저장되어  
브라우저 데이터 삭제 시 목록이 초기화되는 문제를 해결.  
로그인 구현 전까지 `device_id`(익명 UUID) 기반으로 DB에 저장.

### Supabase 테이블 설계

```sql
create table user_exams (
  id         uuid        default gen_random_uuid() primary key,
  device_id  text        not null,
  exam_id    text        not null,
  created_at timestamptz default now(),
  unique (device_id, exam_id)
);
```

- RLS 활성화 + `anon_all` 정책 (로그인 구현 후 정책 교체 예정)
- `device_id` 인덱스 추가 (`idx_user_exams_device_id`)
- 스키마 파일: `supabase/schema.sql`

### 동작 방식

| 상황 | 동작 |
|---|---|
| 앱 시작 | localStorage 즉시 표시 → Supabase에서 최신 목록 동기화 |
| 시험 추가/삭제 | UI 즉시 반영 (낙관적 업데이트) → Supabase INSERT/DELETE |
| 오프라인 / DB 오류 | localStorage 캐시 유지, 다음 접속 시 재동기화 |

### 변경 파일
- `supabase/schema.sql` — 신규 (테이블 DDL)
- `src/hooks/useDeviceId.js` — 복구 (익명 UUID 발급)
- `src/hooks/useMyExams.js` — Supabase CRUD 연동, localStorage 캐시 병행
- `src/pages/My/index.jsx` — `synced` 상태 연결, 동기화 중 안내 문구
- `src/pages/Home/index.jsx` — "시험 추가 →" 링크 `/schedule` → `/my` 수정
